### 이 프로젝트는 뭔가? 
- README.md 에 잘 설명되어 있음 
- Notion 을 대체하기 위한 칸반보드 프로젝트임 
- 궁극적으로 지향하는 바는 Notion 에 준하는 사용감을 사용자들에게 제공하는 것
- 완전 무료, 완전 Private 

### 작업은 어떻게 하니? 
- 시간이 없다. 병렬로 할 수 있는 작업은 최대한 병렬로 처리한다. 
- 모든 계획을 세울때는 현재 프로젝트에서 감당 가능한 수준의 변경인지를 최우선으로 고려한다. 
- 리팩토링의 기준은 AI 이다. AI가 감당 불가능한 수준의 LOC가 될 경우에 code splitting, refactoring 을 고려한다. 
- 코드변경 계획을 세운 후 self-criticize 하는 단계를 최소 2번 이상 거치도록 한다.
- 코드 변경이후 self-review 하는 단계를 최소 1번 이상 거치도록 한다. 기준은 최소한 컴파일이 되는가부터, 공식 문서에 부합하는 수정인가, 유지보수에 적합한 수정인가를 확인한다. correctness 를 우선적으로 검증한 후에 maintainability 에 대한 검증을 수행한다. 
- 새로운 패키지를 사용할 때는 최신 stable 버전 사용을 우선적으로 검토한다. 

### 릴리즈 
- release 는 git tag 를 설정한 후 진행한다 
- 버전은 minor 수정인지, 규모있는 수정인지에 따라서 version number 를 증가시킨다. 
- Major version 1.0.0 이전에는 별도의 테스트를 작성하지 않는다. Major 버전 release 에 대한 결정은 사람이 한다.
